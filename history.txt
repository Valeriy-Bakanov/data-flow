Краткая история разработки программного комплекса DF
======================================================

1).Начало - ноябрь 2009. Выбран пользовательский интерфейс SDI 
(однооконный) в системе Win'32.

2). 18.XI.2009 и 10.XII.2009 - получены первые графики 
интенсивности вычислений (на "миллиметровке").

3).Весна 2011 - статья в СУПЕРКОМПЬЮТЕРЫ #1(5).

4).Лето 2015 - долгая работа над программой решения СЛАУ 
10-го порядка.

5).Лето 2015 - введена система управления приоритетами выполнения 
операторов, написана статья в ПИ (опубликована в сентябре 2015).

6).В начале сентября 2017 (Крым, Чокрак) задействована вЫгрузка 
файлов с сервера Data-Flow (использован компонент из набора Indy).

7).Январь-февраль 2018:
- введено окно выдачи графика интенсивности вычислений (компонент 
Chart из набора C++ Builder 6) 
- разработан макрос работы с 1D-массивами

8).Август-сентябрь 2018 (Крым - Щёлкино, Чокрак): 
- отлажена предикАтная система выполнения операторов (впервЫе ЯПФ
с учётом предикатов получена путём импорта из DF и обработки в SPF
30.VIII.2018 для программы нахождения вещественных и комплексных 
корней полного квадратного уравнения на Чокраке, Крым)
- разработана система цветовой подсвЕтки (и выбора цветОв) ячеек 
таблицы операторов во время выполнения программы

9).Сентябрь-октябрь 2018 - пересмотрена логика OnlySET... 
и Finish... в сторону лучшего понимания и снижения возможных 
ошибок.

10).Октябрь 2018 - для форматного вывода применена безопасная (в 
смысле использования длины строки вывода) функция vsnprintf c 
автоматическим увеличением длины строки при нехватке её).

11).Март 2019 - ужесточена (вывод ShowMessage) реакция 
на ситуацию нехватки памяти данных Mem_Data[][] в функции 
Add_toData().

12).Апрель 2019 - добавлено пользовательское PopUp-меню на окно
вывода лога (протокола) работы симулятора.

13).Май 2019 - формат экспорта ИГА-файлов (.GV) изхменён в 
соответствие со стандартом DOT 
(https://ru.wikipedia.org/wiki/DOT_(%D1%8F%D0%B7%D1%8B%D0%BA).

14).Август 2019 - добавлен Ctrl+F7 для вывода файла *.mvr со
временем выполнения каждого оператора. Версия 4.4.1.

15).Сентябрь 2019 - добавлен Сtrl+F6 вывод графика выполнения 
операторов каждым АИУ.

16).Апрель 2020 - добавлено автоматическое протоколирование
в подкаталог Out!Data.

17).Май 2020 - команды PEQ,PNE,PLT,PGTE,PLE,PGE сделаны 
двухоперандными (так меньше кода писать при сравнении двух 
чисел).

18).Март 2021 - исправлена некорректность задания Max_Proc 
при запуске с командной строкой.

19).Март 2021 - добавлена возможность динамического 
отслеживания выполняемых операторов в таблице машинных
инструкций.

20).Март 2021. Изменены некоторые переменные в DATA_FLOAT.INI:
Max_ -> max_  How_ -> how_

21).Июль 2021 (Щёлкино). Включена обработка случая нулевого 
числа исполняемых операторов. Улучшен (убЫстрен!) код обработки 
ГКВ-операторов. Вывод результатов выполнения предикатных 
операторов сделан более понятным.

22).Август 2021 (Щёлкино). добавлена возможность спекулятивного 
выполнения инструкций ("режим ITANIUM"a).

23).Сентябрь 2021. Устранены проблемы с "перемешиванием" инструкций
и восстановленем активности кнопок после "Останов/сброс".

24).Март 2022 (ver. 4.6). Обмен данными c сервером разработчика
переведён на FTP.

25).Февраль 2023 - введённве в поле "Число АИУ" не нужно запоминать
в "backend" нажатием Enter (сразу можно нажимать F9 или кнопку 
"Выполнение").

26).Май 2023 - новый вариант работы с 1D- и 2D-псевдомассивами 
(расширенные возможности) + longjmp при некорректности синтаксиса 
макроса псевдомассивов. 

27).Июнь 2023 - добавлена возможность использования символа '?' в 
качестве разделителя перед полем предиката в операторах
(теперь допустимы и '?' и ',').

28).Декабрь 2023 - задействована контекстная chm-помощь (пока 
только для главного окна программы).


* Размер выполняемого модуля 1217 кБайт 
(компиляция Release, Pentium, Word).
